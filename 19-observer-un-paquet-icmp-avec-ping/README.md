# Observer un paquet ICMP avec ping

**Rappel théorique** Avant de commencer, revenons sur les concepts essentiels. Un paquet est l'unité de base de communication sur un réseau : c'est un bloc de données encapsulé avec des informations d'en-tête (adresse source, adresse destination, protocole, etc.). Le protocole ICMP (Internet Control Message Protocol) est un protocole de couche réseau (couche 3 du modèle OSI) utilisé principalement pour les diagnostics réseau. Il ne transporte pas de données applicatives, mais des messages de contrôle : « es-tu joignable ? », « le TTL a expiré », etc. La commande `ping` exploite ICMP en envoyant des messages de type Echo Request (type 8) vers une cible, qui répond par des Echo Reply (type 0). C'est l'un des premiers outils utilisés en reconnaissance réseau et en diagnostic. Mental model : imaginez un sonar sous-marin — vous émettez une onde, et vous attendez l'écho. Si l'écho revient, la cible est joignable. Erreurs fréquentes : confondre ICMP avec TCP/UDP (ICMP n'a pas de port), croire que l'absence de réponse ping signifie forcément que la machine est éteinte (un pare-feu peut bloquer ICMP), ou ne pas remarquer les champs TTL et temps de réponse qui sont pourtant très informatifs. Mini-exemple : `ping 8.8.8.8` envoie 4 paquets ICMP Echo Request vers le DNS de Google et affiche le TTL restant et le RTT (Round Trip Time) de chaque réponse. --- **Énoncé de l'atelier** Dans cet atelier, vous allez utiliser la commande `ping` et l'outil de capture réseau **Wireshark** pour observer concrètement ce qu'est un paquet ICMP,...
