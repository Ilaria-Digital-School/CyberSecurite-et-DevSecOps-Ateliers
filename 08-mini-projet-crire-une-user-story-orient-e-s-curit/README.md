# Mini-projet : écrire une user-story orientée sécurité

## Présentation du mini-projet Ce mini-projet clôture la section d'introduction au DevSecOps. Après avoir manipulé des pipelines CI/CD, intégré des points de contrôle sécurité et simulé des incidents, il est temps de formaliser les exigences de sécurité sous forme de user-stories — un outil central dans les méthodologies Agile/DevSecOps. Une user-story orientée sécurité (aussi appelée 'security user story') traduit un besoin de sécurité en langage métier compréhensible par tous les membres d'une équipe (développeur, ops, product owner, pentesteur). Elle permet d'embarquer la sécurité dès la phase de conception ('shift-left security'). ## Contexte du projet Vous travaillez en tant que futur spécialiste DevSecOps intégré dans l'équipe d'un projet e-commerce (le même contexte que l'atelier précédent). Le product owner vous demande de formaliser les exigences de sécurité de l'application sous forme de user-stories prêtes à être intégrées dans le backlog de l'équipe. Votre mission : produire un ensemble cohérent de user-stories orientées sécurité couvrant les principales menaces identifiables sur une application e-commerce, en lien direct avec les étapes du pipeline CI/CD sécurisé. ## Périmètre fonctionnel de l'application e-commerce L'application cible comprend : - Un système d'authentification et de gestion de session utilisateur - Un catalogue produits avec moteur de recherche - Un panier et un tunnel de commande - Un espace client (historique, données personnelles) - Une interface d'administration - Une API REST consommée par une application mobile
