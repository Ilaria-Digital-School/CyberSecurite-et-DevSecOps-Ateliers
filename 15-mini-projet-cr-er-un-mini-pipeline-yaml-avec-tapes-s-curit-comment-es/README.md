# Mini-projet : créer un mini-pipeline YAML avec étapes sécurité commentées

## Description du projet Ce mini-projet constitue le premier jalon concret de votre parcours DevSecOps. Vous allez concevoir et documenter un pipeline CI/CD minimal, écrit en YAML (compatible GitHub Actions ou GitLab CI), intégrant explicitement des étapes de sécurité commentées. Chaque étape du pipeline devra être justifiée du point de vue de la sécurité : pourquoi elle est là, ce qu'elle protège, et à quel moment du cycle de vie elle intervient. Ce projet mobilise l'ensemble des notions abordées jusqu'ici : menaces sur le pipeline, politiques de branche, hooks pre-commit, OWASP Top 10, les trois shifts (Shift-Left, Shift-Right, Feedback Loop), et la logique d'un pipeline sécurisé minimal. ## Contexte professionnel simulé Vous intégrez une petite équipe de développement qui livre une application web. L'équipe n'a aucun pipeline de sécurité en place. Votre mission : proposer un pipeline YAML commenté et explicatif, que des développeurs non-spécialistes en sécurité pourraient comprendre et adopter immédiatement. ## Périmètre technique - Langage cible du pipeline : YAML (GitHub Actions `.github/workflows/security-pipeline.yml` **ou** GitLab CI `.gitlab-ci.yml`) - Application fictive : une application web simple (Node.js, Python Flask, ou autre technologie de votre choix) - Outils de sécurité à intégrer (au minimum 3 parmi) : un linter de sécurité/SAST (ex. Semgrep, Bandit, ESLint security plugin), un outil SCA pour les dépendances (ex. OWASP Dependency-Check, npm audit, Safety), un scanner de secrets (ex.
