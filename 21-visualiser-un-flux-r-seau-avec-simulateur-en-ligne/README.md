# Visualiser un flux réseau avec simulateur en ligne

RAPPEL THÉORIQUE Avant de plonger dans l'atelier, voici les notions essentielles à avoir en tête. Un réseau est un ensemble d'équipements (hôtes, routeurs, switches) reliés entre eux par des liens, qui communiquent en échangeant des paquets. Chaque paquet contient une en-tête (avec adresse IP source, adresse IP destination, protocole, etc.) et une charge utile (les données réelles). Le modèle OSI découpe la communication en 7 couches : Physique, Liaison, Réseau, Transport, Session, Présentation, Application. Dans la pratique, on retient surtout les couches 2 (Ethernet, adresses MAC), 3 (IP, routage) et 4 (TCP/UDP, ports). Un flux réseau est une séquence de paquets échangés entre deux hôtes sur une même connexion logique. Erreurs fréquentes : confondre adresse MAC et adresse IP (l'une est locale au segment, l'autre est globale) ; croire qu'un switch et un routeur font la même chose (le switch opère en couche 2, le routeur en couche 3) ; oublier que chaque paquet est indépendant et peut emprunter un chemin différent. Mini-exemple : quand vous faites un ping vers 8.8.8.8, votre machine génère un paquet ICMP Echo Request (couche 3) encapsulé dans une trame Ethernet (couche 2). Le routeur de votre box reçoit ce paquet, lit l'IP de destination, et le transfère vers Internet. Le serveur distant répond avec un ICMP Echo Reply.
