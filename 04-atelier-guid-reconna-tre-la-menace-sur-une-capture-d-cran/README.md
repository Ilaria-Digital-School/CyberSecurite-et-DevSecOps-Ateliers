# Atelier guidé : reconnaître la menace sur une capture d’écran

## Rappel théorique Avant d'entrer dans le vif du sujet, revenons rapidement sur les concepts essentiels vus dans les activités précédentes. Une **menace** est un événement ou une action potentiellement nuisible qui cible un système, une donnée ou un utilisateur. Une **vulnérabilité** est une faiblesse exploitable qui permet à cette menace de se concrétiser. Le **risque** est la probabilité que cette menace exploite cette vulnérabilité, combinée à l'impact que cela pourrait avoir. Enfin, les **piliers CIA** — Confidentialité, Intégrité, Disponibilité — permettent de qualifier quel aspect de la sécurité est menacé dans chaque situation. Erreur fréquente : confondre la menace et la vulnérabilité. La menace, c'est ce qui pourrait arriver (ex. : un attaquant tente une injection SQL) ; la vulnérabilité, c'est ce qui le permet (ex. : un champ de saisie non filtré). Le risque, c'est la combinaison des deux, pondérée par l'impact. Mini-exemple : une capture d'écran montre une page de login avec le message d'erreur "Utilisateur inconnu". Ce message révèle l'existence ou non d'un compte, ce qui constitue une **fuite d'information** — une menace pour la **confidentialité**. Gardez ce modèle mental en tête tout au long de l'atelier : Menace → Vulnérabilité → Pilier CIA impacté. --- ## Énoncé de l'atelier Dans cet atelier, vous allez analyser une série de captures d'écran représentant des interfaces web ou des réponses HTTP, et identifier les menaces visibles ou sous-jacentes.
