# Atelier : visualiser un pipeline CI/CD simplifié

**Rappel théorique** Avant d'entrer dans le vif du sujet, revenons sur les fondamentaux abordés dans le chapitre précédent sur DevOps et DevSecOps. Le CI/CD (Continuous Integration / Continuous Delivery ou Deployment) est un ensemble de pratiques qui automatise le cycle de vie d'une application, depuis l'écriture du code jusqu'à sa mise en production. Le CI (Intégration Continue) consiste à fusionner fréquemment le code des développeurs dans un dépôt partagé, en déclenchant automatiquement des tests et des vérifications. Le CD (Livraison/Déploiement Continu) prolonge cette logique en automatisant la livraison de l'application vers des environnements de test ou de production. Le mental model à retenir : imaginez une chaîne de montage industrielle. Chaque étape a un rôle précis, et si une pièce est défectueuse, la chaîne s'arrête avant que le produit final ne sorte. Dans un pipeline CI/CD, chaque étape (build, test, analyse de sécurité, déploiement) est un maillon de cette chaîne. En DevSecOps, on insère des contrôles de sécurité (SAST, SCA, DAST) directement dans cette chaîne, pour détecter les failles le plus tôt possible. Erreur fréquente : confondre CI et CD, ou croire que le pipeline se résume à « commit → déploiement ». En réalité, plusieurs étapes intermédiaires critiques existent entre les deux. Mini-exemple : un développeur pousse du code → le pipeline déclenche une compilation, puis des tests unitaires, puis une analyse statique de sécurité (SAST), puis un déploiement sur un environnement de staging, puis des tests dynamiques (DAST), et enfin une mise en production si tout est vert....
